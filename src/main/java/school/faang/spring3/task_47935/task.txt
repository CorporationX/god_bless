Вы — командующий армией героев меча и магии, и вам поручено вычислить общую силу армии, состоящей из разных подразделений: лучники, мечники, маги и т.д. Но армия большая, так что нужно распределить расчёты между несколькими потоками, чтобы быстрее получить результат.

Требования:

1. Создайте классы:

Archer (лучник)

Swordsman (мечник)

Mage (маг)

Если хотите, добавьте другие подразделения.

У каждого бойца должно быть поле power — это его сила. Например, лучник может иметь силу 25, а маг — 50.

2. Напишите класс Army, который будет содержать список всех ваших бойцов. Внутри этого класса напишите метод calculateTotalPower(), который вычисляет общую силу армии.

3. Метод calculateTotalPower() должен использовать распараллеливание. Это значит, что каждый поток будет считать силу для отдельного бойца, а потом все потоки объединят свои результаты. Всё, что нужно делать потоку — это складывать поле power бойцов.

4. Для создания потоков используйте класс Thread. Для удобства, можно создать свой класс, который будет наследовать Thread, чтобы хранить в нём данные для расчётов.

5. Внешняя синхронизация не должна использоваться, только внутренняя механика распараллеливания.

ВАЖНО: если вы уже знакомы с темой синхронизации, то можете использовать. Если же ещё ничего не слышали про слово synchronized или не знаете пока, как правильно им пользоваться, то делайте без него! Цель здесь — именно научиться запускать потоки и распараллеливать работу. Потенциальные проблемы синхронизации мы будем решать позже!

Пример использования:

public class Main {
    public static void main(String[] args) throws InterruptedException {
        Army army = new Army();
        army.addUnit(new Archer(25));   // считаем в потоке 1
        army.addUnit(new Swordsman(40)); // считаем в потоке 2
        army.addUnit(new Mage(50));     // считаем в потоке 3
        army.addUnit(new Mage(20));     // считаем в потоке 4

        int totalPower = army.calculateTotalPower();
        System.out.println("Общая сила армии: " + totalPower);
    }
}
