Sprint04. Task06. CompletableFuture. TwitterSubscriptionSystem
------------

Условия задачи
Представьте, что вам нужно разработать простую модель системы подписок для Twitter. Ваша задача — создать программу на Java, которая будет использовать CompletableFuture для асинхронного добавления подписчиков в учетные записи Twitter, и синхронизировать доступ к общему ресурсу — количеству подписчиков.

Цель:
Создайте программу, которая асинхронно добавляет подписчиков в учетную запись Twitter и корректно синхронизирует доступ к общему ресурсу.

Требования:
1️⃣ Создайте класс TwitterAccount, который представляет учетную запись Twitter. Поля:
username — уникальное имя учетной записи,
followers — количество подписчиков.
Создайте конструктор и методы getters/setters для этих полей.

2️⃣ Создайте класс TwitterSubscriptionSystem, который будет управлять подписками на учетные записи Twitter.

3️⃣ Реализуйте метод addFollower(TwitterAccount account), который увеличивает количество подписчиков учетной записи. Этот метод должен быть синхронизирован, чтобы корректно работать при одновременном доступе из нескольких потоков.

4️⃣ Создайте метод followAccount(TwitterAccount account), который асинхронно добавляет подписчика к учетной записи, вызывая addFollower.

5️⃣ Используйте CompletableFuture для асинхронного запуска задач внутри метода followAccount.

6️⃣ Запустите несколько задач подписки на одну учетную запись, ожидайте завершения всех задач с помощью CompletableFuture.allOf() и выведите количество подписчиков учетной записи.

Почему эта задача полезна?
Эта задача показывает важность синхронизации потоков при работе с общим ресурсом. Когда несколько потоков одновременно изменяют одно и то же состояние (например, количество подписчиков), важно обеспечить правильный доступ к этому ресурсу, чтобы избежать ошибок и некорректных данных.

Это модель поведения реальных приложений, таких как Twitter, где множество пользователей подписываются на один аккаунт одновременно, и система должна корректно увеличивать счетчик подписчиков.
