Условия задачи
Предположим, что мы разрабатываем приложение для сериала Игра Престолов,
где пользователи могут присоединиться к различным домам и бороться за
контроль над Вестеросом. Когда пользователь подключается к дому, он
должен выбрать роль, которую он хочет играть в этом доме: лорд, рыцарь, маг и т.д.

Однако, количество доступных ролей может быть ограничено, и если все роли
уже заняты, пользователь должен ожидать, пока один из игроков не покинет свою роль.

Создайте класс House, который будет представлять дом в Вестеросе.
Добавьте в класс поля, хранящие список доступных ролей и количество доступных ролей.

Создайте класс User, который будет представлять пользователя в приложении.
Добавьте в класс поля, хранящие имя пользователя, дом, в который он присоединился,
и выбранную роль.

Реализуйте метод joinHouse() в классе User, который будет позволять пользователю
присоединиться к дому и выбрать роль. В этом методе пользователь должен сначала
проверить, есть ли доступные роли для дома, и только после этого выбрать свою роль.
Если все роли заняты, пользователь должен ждать, пока один из игроков не покинет свою роль.

Реализуйте метод leaveHouse() в классе User, который будет позволять пользователю
покинуть дом и освободить свою роль.

Реализуйте методы addRole() и removeRole() в классе House, которые будут добавлять
и удалять роли для дома. Когда пользователь выбирает роль, метод addRole() должен
уменьшать количество доступных ролей на 1, а когда пользователь покидает роль,
метод removeRole() должен увеличивать количество доступных ролей на 1.

В методе addRole() добавьте возможность оповестить все потоки о том, что
доступность ролей изменилась.

В методе joinHouse() добавьте блокировку потока, если все роли заняты.
В этом блоке поток должен ожидать, пока один из игроков не покинет свою роль.
Когда пользователь покидает свою роль, метод removeRole() будет вызван, количество
доступных ролей увеличится на 1, и все потоки будут оповещены о том, что роль освободилась.

Добавьте блокировку потока в метод leaveHouse(), чтобы предотвратить изменение
ролей во время выполнения этого метода.

Запустите приложение и проверьте, что все потоки корректно ожидают, когда все роли заняты,
и что пользователи могут выбирать свои роли, когда они становятся доступными.
